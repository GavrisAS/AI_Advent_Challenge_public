# Day 02 — Answer Format

## Исходное условие

🔥 День 2. Формат ответа

Отправьте один и тот же запрос, но:

- добавьте явное описание формата ответа;
- добавьте ограничение на длину ответа;
- добавьте условие завершения ответа (stop sequence или явную инструкцию).

Сравните ответы:

- без ограничений;
- с ограничениями.

Результат: один и тот же запрос с разным уровнем контроля ответа через API.

Формат: видео + код.

## Цель задания

Понять, как формат, лимит длины и stop sequence меняют управляемость ответа модели.

## Реализация

В `snapshot/ANSWER_FORMAT.py` один пользовательский запрос отправляется в нескольких вариантах: с явным форматом, с ограничением длины и с маркером завершения `###END###`. Скрипт выводит содержимое ответа и metadata API, включая `finish_reason` и `usage`, если провайдер их вернул.

## Выводы

Явные ограничения делают ответ модели заметно более предсказуемым. Stop sequence полезен как технический ограничитель, но его лучше сочетать с понятной текстовой инструкцией и проверкой `finish_reason`.
