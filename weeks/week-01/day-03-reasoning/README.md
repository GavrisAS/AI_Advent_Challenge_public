# Day 03 — Reasoning

## Исходное условие

🔥 День 3. Разные способы рассуждения

Возьмите одну задачу (логическую, алгоритмическую или аналитическую) и решите её через API четырьмя способами:

- получите прямой ответ без дополнительных инструкций;
- добавьте в промпт инструкцию: «решай пошагово»;
- попросите модель сначала составить промпт для решения задачи, а затем используйте его;
- создайте в промпте группу экспертов (например: аналитик, инженер, критик) и получите решение от каждого.

Сравните:

- отличаются ли ответы;
- какой способ дал наиболее точный результат.

Результат: несколько решений одной задачи и их сравнение.

Формат: видео + код.

## Цель задания

Сравнить, как разные prompting-подходы влияют на качество рассуждения и итоговый ответ модели.

## Реализация

В `snapshot/REASONING.py` реализованы четыре способа решения одной задачи через DeepSeek API: прямой ответ, пошаговая инструкция, метапромптинг и группа экспертов. После этого выполняется отдельный запрос для сравнения решений и выбора наиболее точного подхода.

Демонстрационная задача:

> Я решил сегодня помыть машину. Автомойка находится в 50 метрах от моего дома. Как лучше, пойти пешком или поехать на машине?

Ожидаемая проверочная идея: поехать на машине, потому что её нужно помыть.

## Выводы

Разные prompting-подходы дают не только разную форму ответа, но и разный уровень самопроверки. Для задач с ловушкой полезнее явно заставлять модель проверять условия или организовывать независимые роли.
