# Day 04 — Temperature

## Исходное условие

🔥 День 4. Температура

Выполните один и тот же запрос с параметрами:

- `temperature = 0`;
- `temperature = 0.7`;
- `temperature = 1.2`.

Сравните ответы по:

- точности;
- креативности;
- разнообразию.

Сформулируйте, для каких задач лучше подходит каждая настройка.

Результат: примеры ответов с разной температурой и выводы по их использованию.

Формат: видео + код.

## Цель задания

Понять практический эффект `temperature` на стабильность, вариативность и творческую свободу ответа модели.

## Реализация

В `snapshot/TEMPERATURE.py` один и тот же запрос отправляется в DeepSeek API с тремя значениями `temperature`: `0`, `0.7` и `1.2`. Скрипт сохраняет ответы и metadata в Markdown-файл для ручного сравнения.

## Как запустить

```bash
cd weeks/week-01/day-04-temperature/snapshot
cp .env.example .env
# Добавьте DEEPSEEK_API_KEY в .env
python TEMPERATURE.py
```

Если вопрос оставить пустым, будет использован демо-запрос про архитектуру сервиса заметок.

## Выводы

Низкая температура лучше подходит для точных и повторяемых ответов. Средняя температура даёт баланс качества и вариативности. Высокая температура полезна для генерации идей, но требует более строгой проверки результата.
